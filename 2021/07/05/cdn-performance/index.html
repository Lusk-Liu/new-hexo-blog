  • Lighthouse插件

主要思路

  • 使用jsDelivr加速GitHub Pages仓库中的使用到的静态资源
  • 开启minify,preconnect等特性加快页面加载

为加速blog访问，修改或启用如下配置：

启用minify

启用minify用于压缩合并JavaScript和CSS文件

修改favicon路径

原路径为GitHub Pages地址，可使用jsDelivr加速

启用preconnect

启用preconnect以在一个 HTTP 请求正式发给服务器前预先执行一些操作，包括 DNS 解析，TLS 协商，TCP 握手

启用lazyload

启用lazyload增加图片懒加载特性以降低FCP

vendors设置

当你使用最新版NexT时请勿修改internal配置

可在此统一修改第三方插件CDN，默认为jsDelivr
