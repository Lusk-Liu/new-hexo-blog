Blog自动化部署

发表于 2021-06-29 更新于 2021-06-30

前期准备

  • 本地安装 node.js
  • 本地安装git
  • GitHub账号
  • GitHub仓库
  • Travis CI账号

本地安装node.js

npm设置阿里源，加快国内访问速度

检查是否设置成功

本地安装git

检查Git是否安装成功

设置Git用户名和邮箱

GitHub账号

本地生成RSA密钥

回车三次

查看rsa公钥信息

将公钥信息添加到GitHub SSH Keys

GitHub仓库

新建一个仓库名称！==用户名的Public仓库

Travis CI

2021-06-15后 应使用 travis-ci.com

  1. 使用GitHub账号登录
  2. 在这里同步账户

hexo

安装hexo

使用terminal/cmd安装

验证hexo安装

使用hexo搭建博客

在当前目录下生成一个名为new-hexo-blog的新目录

new-hexo-blog目录如下

_config.yml：站点配置文件
scaffolds：存放新建文章的模板文件
source ：资源文件夹

在new-hexo-blog目录下运行命令

打开http://localhost:4000即可访问

部署到GitHub Pages

本次使用Travis CI自动部署方案，需要将源文件和生成的网页静态文件放在同一仓库的不同分支，源文件放在main分支，静态文件放在gh-pages分支

GitHub配置

将本地文件推送至GitHub

创建新本地分支gh-pages并关联远程分支

在项目Settings页面Pagestab中选择gh-pages分支

hexo配置

修改new-hexo-blog目录下的hexo的配置文件(_config.yml)

安装hexo-deployer-git

生成网页静态文件并部署

将文件推送至main分支保存，以备份和多设备编辑

使用Travis CI进行自动化部署

注释new-hexo-blog目录下的hexo的配置文件(_config.yml)中repo信息

在new-hexo-blog目录下创建.travis.yml文件，与_config.yml文件同级

Travis正确关联GitHub账号后，这里会出现仓库信息

在这里生成一个新的token以供Travis使用

在travis中对应blog仓库的settings - Environment Variables 中填入token


新建一篇blog，然后推送到GitHub

Travis CI将自动build，等待一会即可完成部署。
